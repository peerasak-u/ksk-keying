รายการ 02โจทย์ — แยก stage เป็น skill แต่ eval มีแค่ interpret

ระบบ ksk-keying ถูกออกแบบให้แต่ละ stage เป็น skill แยก (profile → segment → interpret → link → group → categorize) เหตุผลหลักอย่างหนึ่งคือ เพื่อให้ eval แยกด่านได้ — วัดได้ว่า stage ไหนพังตรงไหน โดยไม่ต้องรันทั้ง pipeline เป็นชั่วโมง

แต่ก่อนเฟสนี้ มีแค่ interpret เท่านั้นที่มี stage eval จริง (tier-A + tier-B, 3 sessions) ส่วน link และ group — ด่านที่ drop เอกสารเคยเกิด (จนต้องมี completeness gate) — ยังไม่มี eval แยกด่านของตัวเอง แม้จะถูกวัดทางอ้อมผ่าน job grader ปลายทางแล้วก็ตาม

สิ่งที่ทำให้เริ่มเฟสนี้ คำถามตรงๆ: "อุตส่าห์แยก stage มาเป็น skill เพื่อ eval ง่ายๆ แต่ทำไปมีแค่ stage 2 เอง — link, group ก็สำคัญ" — ถูกต้อง งานลงทุนไปแล้ว (fixture จาก T12 คือ prerequisite ที่ยากสุด) จึงเก็บเกี่ยวให้ครบในรอบเดียว

รายการ 06T14 — lestrade eval + client-data leak ที่จับได้

lestrade คือ ผู้ตรวจการ exclusion ของ stage 2 — เปิดหน้าที่ watson/marple บอกว่า "ตัดออก (ซ้ำ/ว่าง)" แล้วตัดสิน confirm/refute eval วัดคุณภาพยามตัวนี้เป็น confusion matrix:

ground truth (จากหน้าเอกสาร ไม่ใช่เฉลย)lestrade refutelestrade confirm / เงียบ

10 case = 5 true + 5 seeded-false (รวม 2 adversarial same-seller dup) · ground truth ยืนยันจากอ่านหน้าเอกสารเอง ไม่แตะ answer-key · miss-rate = FN/pos คือตัวเลขความเชื่อใจ (bookable จริงหลุดเงียบ)

Leak ที่จับได้ก่อน merge commit แรกของ opus ใส่ ชื่อผู้ขายจริง + เลขเอกสาร + ยอดเงินจริง ลง committed README (ไฟล์ที่จะขึ้น GitHub) — ผิด hard rule ของ repo ("samples/ gitignored — never commit client data"). parent จับได้ตอน verify → ส่งกลับไป scrub ตาราง + rename case ID ที่ฝังชื่อแบรนด์ → fix แล้ว merged เป็น d7b730e บน main committed files สะอาด 100% (ยืนยันด้วย word-boundary grep)

บทเรียน: dataset เป็น gitignored ก็จริง แต่ เอกสารที่บรรยาย dataset เป็น committed — ต้อง scrub เหมือนกัน

รายการ 07สิ่งที่ยังค้าง — follow-up

รอบนี้ได้ infra + grader + test ครบ แต่ยังไม่ได้เลข live เพราะ stage run ต้องเป็น headless top-level session (ห้ามเป็น subagent) — งานที่เหลือ:

  • Live baselines ทุก stage — รัน claude -p ต่อ stage × N sessions เพื่อได้ agreement/confusion numbers จริง แล้ว pin baseline + ลง scoreboard
  • tier-B expected sets — seam เปิดไว้ dormant ทุกตัว; categorize (เทียบ account-code map ที่ถูก) คุ้มสุดเพราะ account-match 25/84 ในรันจริงชี้ว่านี่คือจุดที่ควรวัด gold
  • lestrade live baseline — dispatch ksk-lestrade จริง ≥3 replicates + human eyeball GROUND-TRUTH ราย case ก่อนเชื่อเป็น measurement
  • agent eval ที่ยังไม่มี — magnum (0), columbo (1), marple (2 report/populate), poirot (5)

คุ้มสุดตัวถัดไป poirot / categorize tier-B — stage 5 เป็นด่านออก และ account-match 25/84 (จริงๆ benign 510110↔510201 swing) ในรันจริงชี้ว่านี่คือจุดที่ควรวัดตรงๆ; grader + seam พร้อมแล้ว เหลือ curate expected + รัน live
